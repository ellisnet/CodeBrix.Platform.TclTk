THIRD-PARTY SOFTWARE NOTICES AND INFORMATION

CodeBrix.Platform.TclTk is a fully managed, cross-platform implementation of the
Tcl scripting language for .NET. It is a port (source-derivative) of the Eagle
project by Joe Mistachkin and the Eagle Development Team; the CodeBrix.Platform.TclTk
assembly incorporates source code derived from that project. The companion
CodeBrix.Platform.TkCanvas library is a managed implementation of the Tk toolkit's
behavior, and CodeBrix.Platform.TclTk.Extras provides managed implementations of
the "sqlite3" (tclsqlite) and "pdf4tcl" Tcl command surfaces.

This file is the one-stop shop for the license and provenance information of ALL
third-party code and assets present in this repository — both source code that
was ported or transcribed and files vendored verbatim. It intentionally does NOT
cover NuGet packages that are merely referenced by the projects in this
repository (their own notices ship with those packages); a referenced package is
covered here only if its source code or assets are actually present in this
repository.

The sections below identify each third-party work, together with the applicable
copyright notices and license terms.

2. Tcl and Tk (the reference distributions)
================================================================================

    Project:      Tcl (Tool Command Language) and Tk (the Tk toolkit),
                  originally created by John Ousterhout
    Source:       https://www.tcl-lang.org/
                  https://core.tcl-lang.org/tcl/  /  https://core.tcl-lang.org/tk/
    License:      Tcl/Tk License (reproduced verbatim below)

    No C or Tcl source file from the reference native Tcl/Tk distribution has
    been copied verbatim into this repository. However, the libraries in this
    repository deliberately and systematically reproduce the behavior of
    Tcl/Tk 8.6:

      * CodeBrix.Platform.TclTk implements the Tcl language (via the Eagle
        project, Section 1, which is itself Tcl-derived), and features that were
        missing upstream (e.g. "binary", "tailcall", "trace", "{*}" expansion)
        were implemented against the reference Tcl documentation and validated
        line-by-line against the real "tclsh" 8.6 interpreter.
      * CodeBrix.Platform.TkCanvas is a from-scratch managed implementation of
        the Tk toolkit's documented and observed behavior — the canvas item
        model, the classic and ttk widget sets, the pack/grid geometry managers,
        the bind/bindtags event model with Tk %-substitution, fonts, photo
        images, the option database, "ttk::style", and the palette/theming
        commands. In specific, noted places, constants and algorithms were
        transcribed from Tk itself: the "tk_setPalette" color-derivation math
        and the exact "tk_bisque" colors from Tk's palette.tcl, the Tk 8.6
        color-name tables (including the TIP 403 color values), the classic
        theme's default resource values, keysym and event names, and the option
        database's precedence rules.
      * The real "tclsh" and "wish" 8.6 binaries were used throughout as
        development-time validation oracles. Outputs captured from real
        Tk 8.6.16 are checked into this repository as test fixtures
        (tests/CodeBrix.Platform.TkCanvas.Tests/Assets/ under BindOracle/,
        CanvasOracle/, LayoutOracle/, ThemingOracle/, and the
        Images/*.data.txt pixel dumps), generated by this repository's own
        capture scripts in tools/layout-oracle/. These fixture files record
        Tk's observed behavior and are treated as Tk-derived data.

    src/CodeBrix.Platform.TkCanvas/Theming/BuiltinThemes.cs carry color palettes

    color themes (the theme-defaults, theme-monokai, theme-monokai-dimmed,
